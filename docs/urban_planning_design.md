# 🏙️ แผนการออกแบบฟีเจอร์ผังเมือง — TRD LEX Urban Planning

## 📌 ภาพรวม

ผนวกระบบตรวจสอบผังเมือง (คล้าย [DPT Landuse Plan](https://landuseplan.dpt.go.th/main/)) เข้ากับระบบที่ราชพัสดุ TRD LEX เพื่อให้ผู้ใช้สามารถ:
1. **คลิกแปลงบนแผนที่** → เห็นสีผังเมืองและข้อกำหนดการใช้ประโยชน์ที่ดินทันที
2. **ค้นหาจากเลขทะเบียน/ที่อยู่** → แสดงผลว่าอยู่ในโซนอะไร ทำอะไรได้/ไม่ได้
3. **ประเมินความเหมาะสม** ของแปลงราชพัสดุสำหรับวัตถุประสงค์ต่างๆ (พาณิชย์/อยู่อาศัย/เกษตร)

> [!IMPORTANT]
> ระบบนี้ออกแบบมาเพื่อ **เสริมมูลค่าให้ตลาดรองสิทธิ์การเช่า** โดยให้ข้อมูลผังเมืองควบคู่กับข้อมูลราชพัสดุ ช่วยผู้ซื้อตัดสินใจได้ดีขึ้น

## 🏗️ สถาปัตยกรรมระบบ

### Phase 1: ข้อมูลพื้นฐานผังเมือง (Database + API)

#### 1.1 Prisma Model ใหม่

```prisma
model ZoningRegulation {
  id             String   @id @default(uuid())
  zone_code      String   @unique  // "ย.1", "พ.2", "ก.3"
  zone_color     String              // HEX color code
  zone_name_th   String              // "ที่อยู่อาศัยหนาแน่นน้อย ประเภท 1"
  zone_name_en   String?             // "Low-Density Residential Type 1"
  category       ZoneCategory        // RESIDENTIAL_LOW, COMMERCIAL, etc.
  
  // ข้อกำหนดการก่อสร้าง
  max_far        Float?              // Floor Area Ratio สูงสุด
  max_osr        Float?              // Open Space Ratio ต่ำสุด
  max_bcr        Float?              // Building Coverage Ratio
  max_height_m   Float?              // ความสูงอาคารสูงสุด (เมตร)
  max_floors     Int?                // จำนวนชั้นสูงสุด
  
  // ข้อกำหนดทั่วไป
  allowed_uses   Json                // ประเภทกิจกรรมที่อนุญาต
  prohibited_uses Json               // ข้อห้าม
  conditions     Json?               // เงื่อนไขพิเศษ
  
  // อ้างอิงกฎหมาย
  legal_ref      String?             // อ้างอิงกฎกระทรวง
  effective_date DateTime?
  
  parcels        ZoningParcelMap[]
  createdAt      DateTime @default(now())
  updatedAt      DateTime @updatedAt
}

model ZoningParcelMap {
  id              String   @id @default(uuid())
  parcel_id       String
  zoning_id       String
  plan_name       String?           // "ผังเมืองรวมเมืองหนองคาย พ.ศ.2556"
  gazette_ref     String?           // เลขที่ประกาศราชกิจจานุเบกษา
  
  parcel          TreasuryParcel    @relation(fields: [parcel_id], references: [id])
  zoning          ZoningRegulation  @relation(fields: [zoning_id], references: [id])
  
  @@unique([parcel_id, zoning_id])
}

enum ZoneCategory {
  RESIDENTIAL_LOW        // เหลือง
  RESIDENTIAL_MEDIUM     // ส้ม
  RESIDENTIAL_HIGH       // น้ำตาล
  COMMERCIAL             // แดง
  INDUSTRIAL             // ม่วง
  RURAL_AGRICULTURE      // เขียว
  CONSERVATION           // เขียวอ่อน
  GOVERNMENT             // ฟ้า
  OPEN_SPACE             // ขาว-เขียว
  SPECIAL                // พิเศษ
}
```

#### 1.2 Backend API Endpoints

```
GET  /api/zoning/regulations             → รายการข้อกำหนดผังเมืองทั้งหมด
GET  /api/zoning/regulations/{zone_code} → รายละเอียดข้อกำหนดตามรหัส
GET  /api/zoning/lookup?lat=&lng=        → ค้นหาโซนจากพิกัด (reverse geocode)
GET  /api/zoning/parcel/{parcel_number}  → ข้อมูลผังเมืองของแปลงที่ดินเฉพาะ
GET  /api/zoning/check-usage?parcel=&purpose= → ตรวจสอบว่าใช้ประโยชน์ได้หรือไม่
POST /api/zoning/feasibility             → วิเคราะห์ความเป็นไปได้เชิงผังเมือง
```

## 🔧 Technology Stack สำหรับฟีเจอร์ใหม่

| ส่วน | เทคโนโลยี | เหตุผล |
|---|---|---|
| Map Engine | Leaflet (ที่มีอยู่) | ต่อยอดจากระบบปัจจุบัน |
| Spatial Query | Ray Casting + PostGIS (อนาคต) | ค้นหาจุดในพื้นที่ polygon |
| Zoning Data | GeoJSON + Prisma DB | เก็บข้อกำหนดใน DB, geometry ใน GeoJSON |
| PDF Export | jsPDF / html2canvas | สร้างรายงานฝั่ง client |
| Color System | CSS Custom Properties | สีผังเมืองเป็น design tokens |

## 💡 จุดเด่นที่ต่างจาก DPT Landuse Plan

| DPT Landuse Plan | TRD LEX Urban Planning |
|---|---|
| ดูข้อมูลผังเมืองอย่างเดียว | **ผสมกับข้อมูลราชพัสดุ + ตลาดรองสิทธิ์** |
| ไม่มีข้อมูลราคา | **แสดงราคาสิทธิ์การเช่าควบคู่** |
| ต้องไปเว็บแยก | **อยู่ในระบบเดียว (one-stop)** |
| ข้อมูลทั่วประเทศแต่ไม่ลึก | **เจาะลึกเฉพาะราชพัสดุ + compliance** |
| ไม่มี feasibility check | **ตรวจสอบความเหมาะสมอัตโนมัติ** |

> [!TIP]
> **จุดขายหลัก:** ผู้ใช้สามารถเห็นทั้งข้อมูลผังเมือง + ราคาสิทธิ์การเช่า + ข้อกำหนด ในหน้าเดียว ช่วยตัดสินใจลงทุนได้เร็วขึ้น

1. **เริ่มจาก Phase ไหนก่อน?** — แนะนำ Phase 1+2 (ข้อมูล + แผนที่สี) เพราะเห็นผลเร็ว
2. **ข้อมูลผังเมืองจริงของหนองคาย** — ต้องหา PDF กฎกระทรวงผังเมืองรวมเมืองหนองคาย มากรอกข้อมูลจริง หรือใช้ข้อมูลจำลองก่อน?
3. **ต้องการดึงข้อมูลจาก DPT API โดยตรงไหม?** — ถ้าเปิดให้ใช้งานจะต่อ API ตรง
